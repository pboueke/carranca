# Docs-Before-Code Policy

Before implementing any feature:
1. Create or update a design document describing what you're building and why
2. Define success criteria and test cases
3. Only then begin implementation

Before committing code:
1. Verify tests exist for all new codepaths
2. Verify documentation is updated
3. Verify the design document reflects what was actually built
